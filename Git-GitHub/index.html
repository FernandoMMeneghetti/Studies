  • Aula 01 - O que é, pra que serve, como funciona?;
    O que é GitHub?
    -É um serviço para controle de versionamento de softwares. Serve para trabalho em grupo nos mesmos arquivos e para controle de alterações.
    O que versionamento de código?
    -É um meio de salvar diferentes estados de um código para ver as diferenças entre uma versão anterior e uma nova (debug), bem como salvar estados para recuperar em caso da nova versão não estar funcionando corretamente (roll-back).
    Palavras chave
    -Commit: Enviar sua versão de um código para avaliação e implementação
  • Aula 02 - Instalando o GIT;
    Código para testar a instalação do GIT:
    No Prompt de comando do Windows digite "git --version". Se aparecer a versão do GIT está tudo certo. Se der erro então nào está instaldo. Sites Git:
    Git (https://git-scm.com/)
    GitHub (https://github.com/)
    Bitbucket (https://bitbucket.org/product/)
    Palavras Chave
    -Repositório: Meu projeto dentro do Git
  • Aula 03 - Configurando o GIT;
    Dentro do CMD do Windows, usar os códigos para gravar ou ler configurações do Git:
    -git config --global user.name "XXX": Salva FXXX como nome do usuário deste Git
    -git config --global user.email "XXX@gmail.com": Salva o e-mail XXX@gmail.com como o e-mail do usuário deste Git
    -git config --global core.editor XXX: Salva XXX como o editor padrão deste Git
    -git config user.name: Lê o nome do usuário registrado neste Git
    -git config user.email: Lê o e-mail cadastrado para este usuário do Git
    -git config core.editor: Lê o nome do software editor configurado neste Git
    -git config --list: Lê todas as configurações salvas
  • Aula 04 - Iniciando um repositório;
    Comandos do Terminal do Windows ensinados nessa aula:
    -cd: mudar de pasta (Ex. "cd c:/Users")
    -dir: Listar os arquivos do diretório atual
    -tree /f: Listar em forma de árvore do diretório atual
    -mkdir: Criar diretório dentro da pasta atual
    -git init: inicia um repositório local dentro da paasta atual
  • Aula 05 - Branch, README e Commit;
    Comandos ensinados:
    -git status: Mostra o status dos commits deste branch
    -git add "XXX": Adiciona o arquivo XXX ao commit atual
    -git add -A: Adiciona todos os arquivos não trackeados ao commit atual
    -git commit -m XXX: Manda o commit atual com o comentário XXX.
    -git log: Mostra o histórico dos commits para o branch atual
    Palavras chave:
    -Branch: Versões de um sistema. A principal costuma ser chamada de "Master"
    -Commit: Enviar suas alterações para aprovação e implementação. Costuma ser enviado com algum comentário para saber qual alteração foi feita e por que.
